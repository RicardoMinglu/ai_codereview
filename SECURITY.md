# 安全策略

## 支持范围

当前以 **`master` 分支最新发布 / tag** 为主进行安全相关评估与修复；历史分支若已不再维护，可能不会回溯修补。

## 报告漏洞（私密）

若你认为发现了**可利用的安全问题**（例如未授权访问敏感数据、远程代码执行、Webhook 绕过、认证绕过等），**请勿**在公开 Issue 中讨论细节。

请任选其一（优先推荐 GitHub 官方渠道）：

1. **GitHub Security Advisories（推荐）**  
   在本仓库页面打开 **Security → Report a vulnerability**（若仓库已启用私有漏洞报告），按表单提交。仅维护者可见。

2. **电子邮件**  
   若无法使用上述入口，可将报告发至 **jushi12138@gmail.com**；邮件主题建议包含 `[ai_codereview-security]` 以便识别。  
   **请勿在邮件正文外张贴 exploit 到公开场合。**

建议在报告中尽量包含：

- 受影响版本或 commit（若已知）
- 复现环境与步骤（最小化）
- 影响说明（机密性 / 完整性 / 可用性）
- 是否已有补丁思路（可选）

我们会在合理时间内阅读并回复；严重问题会优先处理。公开披露前通常会与报告者协调修复与披露时间。

## 非安全事项

配置误用、功能建议、一般 Bug 请使用普通 **Issue**，不必走本流程。

## 加固提示（自托管）

自托管部署时，请务必：

- 配置 **`github.webhook_secret`**，校验 GitHub Webhook 签名；
- **报告 Web UI 无内置登录**：不要无防护暴露公网；建议内网、VPN，或通过反向代理做 **HTTPS + 鉴权 / IP 限制**；
- 妥善保管 GitHub Token 与各 AI / 通知渠道密钥，使用密钥管理或环境隔离。

更多说明见 README「安全说明」。
